Resoconto del laboratorio — 22/09/2025

Prof. Marco Cocciniglia — Sintesi di quanto svolto oggi a lezione.

1) Punto di partenza

Abbiamo scaricato e usato un progetto Maven preparato dal docente e creato con JDK 17.

2) Il problema riscontrato

In laboratorio era installato solo JDK 11 e non avevamo i permessi per aggiornare. Risultato: il progetto, così com’era, non compilava.

3) Le due opzioni possibili

  • Opzione A — Installare JDK 17: soluzione ideale, ma non possibile senza permessi.
  • Opzione B — Retarget a Java 11: abbassare la versione di compilazione del progetto a 11.
Perché serve toccare il POM?
Con i progetti Maven la versione del linguaggio/bytecode è decisa dal file pom.xml (non solo dalle impostazioni dell’IDE).
Se il POM chiede release 17 ma la macchina ha JDK 11, Maven segnala l’errore “release version 17 not supported”.

4) Cosa abbiamo fatto (retarget a 11)

Abbiamo modificato il pom.xml per compilare verso Java 11. Ecco il cambiamento minimo:

<properties>
  <maven.compiler.release>11</maven.compiler.release>
  <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
</properties>

Per essere più espliciti e allineare il plugin del compilatore, si può (consigliato) aggiungere anche:

<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-compiler-plugin</artifactId>
      <version>3.11.0</version>
      <configuration>
        <release>11</release>
      </configuration>
    </plugin>
  </plugins>
</build>
Conseguenze del retarget
Dal momento che ora il progetto è “Java 11”, non possiamo usare funzionalità introdotte dopo l’11 (esempi: switch expressions, text blocks, records, sealed classes). In caso compaiano nel codice, vanno riscritte o rimosse.

5) Import del progetto e verifica

  1. Import in IDE (NetBeans/Eclipse) come progetto Maven dalla cartella con il pom.xml.
  2. Selezione della piattaforma JDK 11 nell’IDE (Project Properties / Installed JREs / Project SDK).
  3. Reload/Update del progetto Maven, quindi Clean & Build o Run.

Esito: i progetti hanno compilato ed eseguito correttamente con JDK 11.

6) Ripasso veloce: IDE & Project Explorer

  • IDE: ambiente integrato che ci aiuta a scrivere, compilare, eseguire e fare debug.
  • Project Explorer: pannello che mostra cartelle, file e struttura Maven (src/main/java, src/test/java, ecc.).
  • In Eclipse: Window → Show View → Project Explorer (o Reset Perspective per ripristinare il layout).

7) Paradigma e regole base di Java

  • Java è orientato agli oggetti: progettiamo con classi, oggetti, metodi.
  • Il programma parte dal metodo public static void main(String[] args).
  • Una classe public per file e il file deve chiamarsi come la classe (es. App.java contiene public class App).
  • La cartella deve rispecchiare il package (es. it.scuola...base.es01_hello → cartelle con lo stesso percorso).

8) Cosa portate a casa da oggi

  • Capire che la versione Java è parte della configurazione di progetto (POM).
  • Saper retargettare un progetto Maven a una versione inferiore quando l’ambiente non è aggiornabile.
  • Importare, compilare ed eseguire un progetto Maven in un IDE scolastico.
